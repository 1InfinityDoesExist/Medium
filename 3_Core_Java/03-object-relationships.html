Object Relationships in Java

Examples from real life

⚡ In a nutshell — The Is-a relationship (built with inheritance) · The Has-a relationship (one object used inside another) · Association, and its two flavors: Aggregation (weak) and Composition (strong)

Objects in a program rarely live alone — they are connected to each other, just like things in the real world. Java describes these connections with a few simple relationship names.

Is-a Relationship

  • The Is-a relationship is achieved through inheritance.
  • Example: a Dog is an Animal.
  • Inheritance forms an is-a relation between the parent class and the child class.

The easy test: say the sentence out loud — "X is a Y". If it sounds true (a Dog is an Animal, a Car is a Vehicle), then X should be a child class of Y.

class Animal {                 // parent
    public void eat() {
        System.out.println("This animal is eating.");
    }
}

class Dog extends Animal {     // Dog IS-A Animal
    public void bark() {
        System.out.println("Woof! Woof!");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat();    // inherited from Animal, because Dog IS-A Animal
        d.bark();
    }
}

Output:

This animal is eating.
Woof! Woof!

Remember: Is-a = inheritance (extends). If "X is a Y" sounds natural, X extends Y.

Has-a Relationship

  • Whenever an object is used inside another class, it is called a HAS-A relationship.
  • The relationship could be one-to-one, one-to-many, or many-to-many.

Instead of inheriting, the class simply keeps the other object as a field — like a school keeping a list of its students.

  • A School has Students (one-to-many: one school, many students).
  • A Bike has an Engine (one-to-one: one bike, one engine).
  • A School has Classes (one-to-many).

The easy test: say "X has a Y". If it sounds true (a Bike has an Engine), then class X should contain a Y object inside it.

class Engine {
    public void start() {
        System.out.println("Engine started.");
    }
}

class Bike {
    Engine engine = new Engine();   // Bike HAS-A Engine (one-to-one)

    public void ride() {
        engine.start();             // Bike uses the Engine object it holds
        System.out.println("Bike is moving.");
    }
}

public class Main {
    public static void main(String[] args) {
        Bike bike = new Bike();
        bike.ride();
    }
}

Output:

Engine started.
Bike is moving.

Remember: Is-a = "Dog is an Animal" → inheritance. Has-a = "Bike has an Engine" → one object kept as a field inside another class.

Association

Association is a relationship between 2 different objects. It tells us that the objects are connected. Association comes in two flavors, depending on how tightly their lives are tied together:

  1. Aggregation — the loose connection
  2. Composition — the tight connection

3a. Aggregation (weak relation)

  • Both objects can survive individually — the ending of one object will not end the other object.
  • Aggregation is also known as a weak relation.

Everyday picture: a School and its Students. If the school shuts down, the students still exist — they simply join another school. Their lives are independent.

class Student {
    String name;

    Student(String name) {
        this.name = name;
    }
}

class School {
    String schoolName;
    Student student;                       // Aggregation: student is created OUTSIDE

    School(String schoolName, Student student) {
        this.schoolName = schoolName;
        this.student = student;            // school only "uses" an existing student
    }
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student("Ravi");   // student exists on his own
        School school = new School("City School", s);

        school = null;                     // the school "ends"...
        System.out.println(s.name + " still exists!");   // ...but the student survives
    }
}

Output:

Ravi still exists!

3b. Composition (strong relation)

  • The ending of one object will end the other object.
  • Composition is also known as a strong relation.

Everyday picture: a House and its Rooms. A room cannot exist without its house — if the house is demolished, its rooms are gone too. Their lives are tied together.

class Room {
    String type;

    Room(String type) {
        this.type = type;
    }
}

class House {
    // Composition: the Room is created INSIDE the House.
    // Nobody outside holds it, so when the House ends, the Room ends too.
    private Room bedroom = new Room("Bedroom");

    public void show() {
        System.out.println("This house has a " + bedroom.type + ".");
    }
}

public class Main {
    public static void main(String[] args) {
        House house = new House();
        house.show();

        house = null;   // the house ends -> its bedroom is gone with it
        System.out.println("House destroyed, and its rooms are destroyed with it.");
    }
}

Output:

This house has a Bedroom.
House destroyed, and its rooms are destroyed with it.

Aggregation vs Composition — Quick Comparison

  • Strength — Weak relation — Strong relation
  • Can both objects survive individually? — Yes — ending one does not end the other — No — ending one ends the other
  • Everyday example — School and Students — House and Rooms
  • Who creates the inner object? — Created outside, passed in — Created inside the owner class

Remember: Aggregation = weak = both survive alone. Composition = strong = they live and die together.

Quick Revision

  • Is-a relationship → achieved through inheritance (Dog is an Animal); it connects parent and child classes.
  • Has-a relationship → an object used inside another class (School has Students, Bike has Engine, School has Classes); can be one-to-one, one-to-many, or many-to-many.
  • Association → a relationship between 2 different objects, with two flavors:
  • Aggregation → weak relation; both objects can survive individually — ending one does not end the other.
  • Composition → strong relation; ending one object ends the other.
