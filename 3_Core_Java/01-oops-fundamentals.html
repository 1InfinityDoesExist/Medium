Two Ways of Writing Programs

Before Java, most programs were written in a style called procedural programming. Java uses a newer style called Object-Oriented Programming (OOPs). Let's understand both.

In procedural programming, a program is divided into small parts called functions. Think of it like a long recipe: step 1, step 2, step 3... where every step can touch every ingredient.

The problems with this style:

  • The program is divided into parts called functions.
  • It does not provide a proper way to hide data. It gives importance to functions, and data moves around freely — any function can read or change any data. That makes big programs risky and hard to maintain.
  • Overloading is not possible (you cannot have two functions with the same name doing slightly different jobs).
  • Inheritance is not possible (you cannot build a new piece of code on top of an old one).
  • Code reusability is not present — you often end up copying and pasting code.
  • Examples of procedural languages: Pascal, C, Fortran.

Object-Oriented Programming (OOPs — the Java style)

In OOP, a program is divided into objects. An object keeps its data and the functions that work on that data together in one box. Think of a team of workers: each worker (object) has their own tools (data) and their own skills (functions), and nobody else touches their tools directly.

  • The program is divided into objects.
  • Objects provide data hiding — OOP gives importance to data and protects it.
  • Overloading is possible (same method name, different versions).
  • Inheritance is possible (a new class can reuse an existing class).
  • Code reusability is present — write once, reuse many times.
  • Examples of OOP languages: Java, C#, Python, C++ etc.

Procedural vs OOPs — Quick Comparison

  • Program is divided into — Functions — Objects
  • Data hiding — No proper way to hide data; data moves freely — Objects provide data hiding
  • Importance is given to — Functions — Data
  • Overloading — Not possible — Possible
  • Inheritance — Not possible — Possible
  • Code reusability — Not present — Present
  • Example languages — Pascal, C, Fortran — Java, C#, Python, C++

Remember: Procedural programming thinks in steps (functions). OOP thinks in things (objects). Java is an object-oriented language.

What is an Object?

An object is any "thing" in your program. Every object has exactly 2 things:

  1. Properties / State — what the object has (its data).
  2. Behavior / Function — what the object does (its actions).

Real-life example: a Dog

A Dog is an object because it has:

  • Properties like: Age, Breed, Color etc. (these describe the dog)
  • Behaviors like: Bark, Sleep, Eat etc. (these are things the dog can do)

You can apply this test to anything: a car (properties: color, speed; behaviors: start, brake), a phone (properties: brand, battery; behaviors: call, ring). If it has properties and behaviors, it can be modeled as an object.

What is a Class?

A class is the blueprint of the object.

A class is like a house blueprint; objects are the actual houses built from it. The blueprint itself is not a house — it only describes what every house of that design will look like. In the same way, a class describes the properties and behaviors, and objects are the real things created from that description.

Key points:

  • To create an object, a class is required.
  • The class provides the template or blueprint from which objects can be created.
  • From one class, we can create multiple objects (one blueprint → many houses).
  • To create a class, use the keyword class.

Creating a class in Java

class Student {
    // Properties / state  -> called "data variables"
    int age;
    String name;

    // Behavior / function -> called a "data method"
    public void updateName() {
        name = "Updated Name";
    }
}

Note: A data method works on the data variables (the properties). Here updateName() changes the name variable.

Creating an object in Java

We create an object using the new keyword:

Student engStudent = new Student();

Read it like this: "Make a new Student from the Student blueprint, and call it engStudent."

Complete runnable example

class Student {
    int age;          // property
    String name;      // property

    public void introduce() {          // behavior
        System.out.println("Hi, I am " + name + " and I am " + age + " years old.");
    }
}

public class Main {
    public static void main(String[] args) {
        // One class, multiple objects:
        Student engStudent = new Student();   // object 1
        engStudent.age = 20;
        engStudent.name = "Ravi";

        Student medStudent = new Student();   // object 2
        medStudent.age = 22;
        medStudent.name = "Priya";

        engStudent.introduce();
        medStudent.introduce();
    }
}

Output:

Hi, I am Ravi and I am 20 years old.
Hi, I am Priya and I am 22 years old.

Notice: both objects came from the same class, but each object keeps its own copy of the properties (age, name).

Quick Revision

  • Procedural programming divides a program into functions; data moves freely, no data hiding, no overloading, no inheritance, no reusability (e.g. Pascal, C, Fortran).
  • OOP divides a program into objects; it gives importance to data, supports data hiding, overloading, inheritance, and code reuse (e.g. Java, C#, Python, C++).
  • An object has 2 things: properties/state (age, breed, color) and behavior/functions (bark, sleep, eat).
  • A class is the blueprint of an object — like a house blueprint vs the actual houses.
  • Create a class with the class keyword; create an object with the new keyword: Student s = new Student();
  • One class → many objects, each with its own copy of the properties.
