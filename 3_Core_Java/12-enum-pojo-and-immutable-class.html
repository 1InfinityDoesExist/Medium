POJO — Plain Old Java Object

POJO stands for Plain Old Java Object. It is the simplest kind of Java class — just a plain container for data, with nothing fancy attached.

A POJO:

  • Is an ordinary class — it does not extend any framework class and does not implement any framework interface. No special parent required.
  • Has private fields to hold its data.
  • Has public getters and setters to read and change those fields.
  • Usually has a no-argument constructor.
  • Has no special annotations or rules forced on it.

Analogy: a POJO is like a plain cardboard box with labels — it just holds your data (name, age, ...) neatly, and the getters/setters are the labels through which you look inside or put things in. Frameworks like Spring and Hibernate love POJOs precisely because they are so simple to work with.

// A classic POJO: plain fields + getters + setters. Nothing else.
public class Student {
    private String name;      // private data
    private int age;

    public Student() {        // no-arg constructor
    }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

class Main {
    public static void main(String[] args) {
        Student s = new Student();
        s.setName("Ravi");
        s.setAge(21);
        System.out.println(s.getName() + " is " + s.getAge() + " years old");
    }
}

Output:
Ravi is 21 years old

Remember: POJO = Plain Old Java Object — private fields + public getters/setters + no framework baggage.

Enum

An enum is a special class used when you have a fixed set of constants — days of the week, directions, order statuses, etc.

  • It is a collection of CONSTANTS.
  • Its constants are static and final implicitly — you do not write those keywords; Java adds them for you.
  • It cannot extend any class, because internally every enum already extends java.lang.Enum (and Java allows only one parent). This extends Enum happens by default — you never write it yourself.
  • It can implement interfaces.
  • It can have variables, constructors and methods — an enum is much more than a list of names!
  • It cannot be instantiated with new, because its constructor is always private. Even if you write a default (package-private) constructor, in the bytecode Java makes it private.
  • No other class can extend an enum class. (Think of an enum like a final class — and a final class cannot be inherited.)
  • It can have abstract methods — but then every constant must implement that abstract method.

A basic enum + the four famous methods

The Enum class gives every enum some famous ready-made methods:

  • values() — Returns an array of all constants of the enum — perfect for looping
  • ordinal() — Returns the position (index) of the constant, starting from 0
  • valueOf("NAME") — Converts a String into the matching constant (name must match exactly)
  • name() — Returns the name of the constant as a String
public enum CustomEnum {     // internally: extends java.lang.Enum (default)
    MONDAY,                  // ordinal 0   — implicitly public static final
    TUESDAY,                 // ordinal 1
    WEDNESDAY,               // ordinal 2
    THURSDAY,                // ordinal 3
    FRIDAY                   // ordinal 4
}

class Main {
    public static void main(String[] args) {
        // values() -> loop over every constant
        for (CustomEnum sample : CustomEnum.values()) {
            System.out.println(sample.name() + " -> " + sample.ordinal());
        }

        // valueOf() -> String to constant (exact name required)
        CustomEnum myEnum = CustomEnum.valueOf("FRIDAY");
        System.out.println(myEnum.name());

        // CustomEnum c = new CustomEnum();  // ERROR! constructor is private
    }
}

Output:
MONDAY -> 0
TUESDAY -> 1
WEDNESDAY -> 2
THURSDAY -> 3
FRIDAY -> 4
FRIDAY

Enum with variables, constructor and methods

The constructor runs once for each constant, automatically. It is private (even if you don't say so):

public enum Coffee {
    SMALL(100), MEDIUM(150), LARGE(200);   // constants call the constructor

    private final int price;               // enums can have variables

    Coffee(int price) {                    // private by default —
        this.price = price;                // even in bytecode it is private!
    }

    public int getPrice() {                // enums can have methods
        return price;
    }
}

class Main {
    public static void main(String[] args) {
        for (Coffee c : Coffee.values()) {
            System.out.println(c.name() + " costs Rs." + c.getPrice());
        }
    }
}

Output:
SMALL costs Rs.100
MEDIUM costs Rs.150
LARGE costs Rs.200

Enum implementing an interface

An enum cannot extend a class, but it can implement interfaces:

interface Describable {
    String describe();
}

public enum Direction implements Describable {
    NORTH, SOUTH;

    @Override
    public String describe() {
        return "Direction: " + name();
    }
}

class Main {
    public static void main(String[] args) {
        System.out.println(Direction.NORTH.describe());
    }
}

Output:
Direction: NORTH

Enum with an abstract method

If an enum declares an abstract method, every single constant must implement it (each constant gets its own body):

public enum Operation {
    ADD {
        @Override
        public int apply(int a, int b) { return a + b; }
    },
    MULTIPLY {
        @Override
        public int apply(int a, int b) { return a * b; }
    };
    // every constant MUST implement this:
    public abstract int apply(int a, int b);
}

class Main {
    public static void main(String[] args) {
        System.out.println(Operation.ADD.apply(2, 3));
        System.out.println(Operation.MULTIPLY.apply(2, 3));
    }
}

Output:
5
6

Remember: enum constants are implicitly static final; the constructor is always private; an enum already extends java.lang.Enum, so it cannot extend anything else — and nothing can extend it. A final class cannot be inherited, and an enum behaves the same way.

Immutable Class

Immutable means: once an object is created, its value can never be changed. Any "change" must create a new object instead.

Real examples from the JDK: String and the wrapper classes (Integer, Double, Boolean, ...). When you do s = s + "x", Java builds a brand-new String — the old one never changes.

Why bother? Immutable objects are automatically thread-safe, safe to share, safe to cache, and safe to use as HashMap keys — because nobody can secretly modify them.

The rules for building an immutable class

  1. Declare the class as final — so no subclass can be created that might break immutability. (A final class cannot be inherited.)
  2. Make all class members (fields) private — so nobody outside can touch the data directly.
  3. Also make the fields final — each field is initialized only once, through the constructor, and can never be reassigned.
  4. Do NOT provide any setter methods — setters exist to change values, and change is exactly what we forbid.
  5. Provide only getter methods — and for mutable members (like a List), the getter must return a COPY of the member, not the original (a "defensive copy"). Otherwise the caller could modify your internal list through the reference!

The MyImmutableClass example

import java.util.ArrayList;
import java.util.List;

final class MyImmutableClass {                 // rule 1: final class
    private final String name;                 // rules 2+3: private final
    private final List<Object> petNameList;

    MyImmutableClass(String name, List<Object> petNameList) {
        this.name = name;                      // rule 3: set once, in the
        this.petNameList = petNameList;        //         constructor only
    }

    // rule 4: NO setters anywhere!

    public String getName() {                  // rule 5: getters only
        return name;                           // String is already immutable
    }

    public List<Object> getPetNameList() {
        // rule 5: return a COPY, not the real list (defensive copy)
        return new ArrayList<>(petNameList);
    }
}

class Main {
    public static void main(String[] args) {
        List<Object> pets = new ArrayList<>();
        pets.add("Tommy");
        pets.add("Sheru");

        MyImmutableClass obj = new MyImmutableClass("Uttam", pets);

        // Try to hack the object through the getter:
        obj.getPetNameList().add("Hacker Dog");   // modifies only the COPY

        System.out.println(obj.getName());
        System.out.println(obj.getPetNameList()); // original is untouched!
    }
}

Output:
Uttam
[Tommy, Sheru]

Notice the trick: the caller added "Hacker Dog", but only to the copy returned by the getter. The object's real list stayed [Tommy, Sheru] — that is the power of the defensive copy.

Immutable class checklist

  • Class is final — Nobody can extend it and break the rules
  • Fields are private — No direct access from outside
  • Fields are final, set once in the constructor — Value fixed at birth
  • No setter methods — Nothing can be changed later
  • Getters return copies of mutable fields — The original data can never leak out

Quick Revision

  • POJO = Plain Old Java Object: private fields + public getters/setters + no framework parent. The simplest data-holder class.
  • Enum = a collection of constants that are implicitly static final.
  • An enum internally extends java.lang.Enum, so it cannot extend any other class; it can implement interfaces; no class can extend an enum.
  • An enum can have variables, constructors and methods; the constructor is always private (even in bytecode), so new is impossible.
  • An enum can declare abstract methods — every constant must then implement them.
  • Famous enum methods: values() (all constants), ordinal() (position from 0), valueOf("NAME") (String → constant), name() (constant → String).
  • A final class cannot be inherited.
  • Immutable class = value fixed forever: final class, private final fields, constructor-only initialization, no setters, getters return defensive copies. Examples: String, wrapper classes.
